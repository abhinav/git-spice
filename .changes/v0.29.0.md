## <a name="v0.29.0">v0.29.0</a> - 2026-05-25

This release makes significant changes to how
`gs repo sync`, `gs branch delete`, and `gs branch onto` handle
rebasing branches above deleted or moved branches.
Previously, these would rebase branches right above the affected branches,
but branches further up the stack would be left alone.
This was inconsistent and undesirable.
Now, they *by default* do not rebase upstack branches,
and only update their metadata to point to the new bases.
You can now use the `--restack` flag with all three commands
to restack all upstack branches of affected branches,
or `--restack=aboves` to restack only the branches directly above
(the previous default behavior).

### Added
- Set up an official Ubuntu PPA for downloading git-spice.
- submit: Add 'spice.submit.label.addWhen' configuration option to control when configured labels are added to CRs.
- downstack: Add restack command for the current branch and branches below it
- commit fixup: Add --edit flag to allow editing commit messages.
- log: change IDs in terminal output are clickable OSC 8 hyperlinks.
- branch delete: Add --restack option to control restacking of branches above deleted branches.
  Use the 'spice.branchDelete.restack' configuration option to change the default.
- branch onto: Add --restack option to control restacking of branches above moved branches.
  Use the 'spice.branchOnto.restack' configuration option to change the default.
### Changed
- repo sync: Branches above deleted branches are now retargeted without being restacked unless --restack is used.
- branch delete: Branches above deleted branches are now retargeted without being restacked unless --restack is used.
- branch onto: Branches above moved branches are now retargeted without being restacked unless --restack is used.
- repo sync: --restack option now runs in 'upstack' mode: all upstack branches of deleted merged branches will be restacked. Override with --restack=aboves to restack only the branches directly above, or --restack=none to disable restacking (this is the default behavior). Change the default by setting the 'spice.repoSync.restack' configuration option.
- submit: Skip submission when a downstack base's change has been merged (use --force to bypass).
- navigation: Allow up, down, top, and bottom to navigate from a detached HEAD when it points at exactly one local branch.
### Fixed
- bitbucket: Preserve PR description when adding reviewers
- commit fixup: Fix unstaged worktree changes when staged fixup changes conflict.
- navigation: Non-positive branch counts for up and down are now rejected instead of causing a panic.
- log: Avoid warning about unavailable remote metadata when listing local-only repositories.
- top: Fix --detach not being honored when already on the top-most branch.
- restack: Branches retargeted without rebasing are now correctly detected as needing restack when their old base remains in history.
- repo sync: Fixed retargeted branches replaying already-present downstack commits during restack.
- repo sync: Upstack PRs are now retargeted after their base branch is merged.
  This handles cases when the forge doesn't automatically retarget their bases.
