## <a name="v0.25.0">v0.25.0</a> - 2026-04-06
### Added
- bitbucket: Add Bitbucket Cloud forge support
- auth: Add Git Credential Manager (GCM) authentication support for GitHub and Bitbucket
- submit: Add 'spice.submit.skipRestackCheck' configuration option to skip restack verification during branch submission. This can be configured to skip the check only for branches on top of trunk, or for all branches.
- branch diff: Show diff between a branch and its base
- log: Add option to display comment and resolution counts in listings.
### Changed
- branch checkout: Add 'spice.branchCheckout.trackUntracked' configuration option to control tracking of untracked branches. Defaults to prompting the user, but can be set to 'always' to always track untracked branches, or 'never' to not track nor prompt.
- Guidance printed by git-spice commands will use the name of the active binary instead of always using 'gs' or 'git-spice'.
- Upgrade TUI rendering to v2 of underlying libraries. No user-facing changes are expected, but significant internal refactoring was required. If you encounter unexpected side effects, please [report them here](https://github.com/abhinav/git-spice/issues).
- gitlab: Replace the GitLab API client with a smaller internal client. This allows us to reduce the binary size by >25%. We expect no other user-facing impact from this, so please [report bugs](https://github.com/abhinav/git-spice/issues) if you see any.
### Deprecated
- branch checkout: Deprecate 'spice.branchCheckout.trackUntrackedPrompt' configuration option in favor of 'spice.branchCheckout.trackUntracked'.
### Removed
- Remove 'gs' binary from published packages: GitHub Releasees, Homebrew Cask, and ArchLinux AUR package.
- A warning is no longer printed if the program is invoked as 'gs'.
### Fixed
- restack: Fix bug where commits from a branch could be dropped during restack if they showed up in the base branch's reflog.
- Branch selection prompt now scrolls when the terminal is too short to show all matching branches.
- submit: Honor worktree-local and repository-local Git editor settings for interactive prompts.
- gitlab: Accept dots in glab CLI authentication tokens
- commit fixup: Preserve unstaged worktree changes when rejecting an empty index.
- restack: Failures during an internal `git ls-files` check are now reported with a clearer error message instead of an unhelpful unmerged-file report.
