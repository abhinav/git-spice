## <a name="v0.30.0">v0.30.0</a> - 2026-06-29

The big news in this release is the [`merge` experiment](https://abhinav.github.io/git-spice/cli/experiments/#merge) enabling the following commands: `gs downstack merge`, `gs stack merge`, and `gs branch merge`.
These turn git-spice into a local merge queue, allowing you to merge stacks of branches in a single automated operation.

![demo of a stack merge](https://media.githubusercontent.com/media/abhinav/git-spice/refs/heads/main/doc/src/img/2026-05-29-merge-progress-demo.gif)

Enable the experiment with `git config spice.experiment.merge true`, try it out, and [share your experience here](https://github.com/abhinav/git-spice/discussions/1327).

The other major highlight in this release is support for new forges: Gitea and Forgejo. Forgejo support also enables use of git-spice with [Codeberg](https://codeberg.org/).

### Added
- Add 'gs branch merge', 'gs stack merge', and 'gs downstack merge' commands to merge stacks of branches.
- forge: Add Forgejo and Codeberg support.
- forge: Add Gitea support.
- forge: The new 'spice.forge.kind' configuration option allows explicitly specifying the forge kind when automatic detection isn't satisfactory.
- commit create, commit amend, branch create: The -m/--message flag may now be passed multiple times to add multiple paragraphs to the commit message, matching 'git commit'.

### Changed
- commit amend: Prompt before amending when a rebase stopped to resolve a conflict. This prevents accidentally merging a conflicting commit into its parent.
- down: Improved performance when navigating multiple branches down a stack.
- github: GitHub user ID lookups during the same operation are now reused, avoiding repeated API calls.

### Deprecated
- submit: Deprecate the 'spice.submit.label' option in favor of 'spice.submit.labels'. The deprecated form will be deleted in a future release.
- submit: Deprecate the 'spice.submit.label.addWhen' option in favor of 'spice.submit.labels.addWhen'. The deprecated form will be deleted in a future release.

### Fixed
- submit: Branches submitted with --no-publish can still be updated with --update-only.
- repo sync: Detect manually submitted merged changes when a source branch name is reused by a newer open change.
- Custom shell aliases now have access to standard input. This previously prevented use of interactive git-spice commands in custom shell aliases.
- branch checkout: Branches checked out in other worktrees can now be selected with --dry-run.
- branch track: Reject revision names such as HEAD when tracking a branch.
- Navigation commands ('up', 'down', 'top', 'bottom') no longer print restack warnings with --dry-run.
- branch submit: Fix accidental pushes to trunk when a branch's upstream is trunk.
- Captured subprocess output is now limited to prevent unbounded memory use.
