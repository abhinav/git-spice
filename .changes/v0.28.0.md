## <a name="v0.28.0">v0.28.0</a> - 2026-05-03

This release of git-spice adds support for fork-based workflows, for contributors who do not have write access to the upstream repository. This is a common way to contribute to open-source projects.

In fork mode, git-spice pushes submitted branches to your fork, opens Change Requests against the upstream repository, and synchronizes trunk from upstream. Change Requests are created only for branches based directly on trunk; other stacked branches are still pushed to your fork.

Read more at [Fork workflows](https://abhinav.github.io/git-spice/guide/cr/#fork-workflows).

### Added
- Support fork mode. Change Requests are opened against the upstream remote for trunk-based branches, and submitted branches are pushed to the push remote.

### Changed
- storage: Change repository storage to support fork mode. Repositories using fork mode cannot be opened with older verions of git-spice.

### Fixed
- repo sync: Prevent branches with unsubmitted local commits from being deleted after their change request is merged.
- repo sync: Fix spurious conflicts after squash-merging branches with upstack work.
- submit: Fix --update-only submissions failing instead of skipping unsubmitted branches that need to be restacked.

### Removed
- Remove Snap release. This was not successful in the prior release.
